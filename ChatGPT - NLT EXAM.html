  • Vanishing gradients (p.85).

(File: Lecture 10 - Attention & Transformers.pdf)
Lecture 10 - Attention Transf…

✅ Summary

This lecture explains attention mechanisms, the evolution from RNNs → RNNs with attention → Transformers, and details Transformer components.

1. Encoder–Decoder in Machine Translation (MT)

Pages 3–11 show how RNN-based MT systems:

  • Encode a source sentence into a single vector representation.

  • Decode it word-by-word into target language.

2. Problems with RNN Encoders

Long sentences deteriorate representation quality (example on p.14).

3. Bahdanau Attention

The lecture introduces why attention is needed (p.15–20):

  • Instead of passing a single sentence vector, the decoder receives weights over all encoder hidden states.

  • Attention weights ≈ “which source words are relevant for generating this target word”.

Mathematically:

  • Each decoder step computes a context vector (p.24–28).

  • Weights are softmax-normalised alignment scores.

4. General Attention Mechanism

Pages 35–53 explain attention generically:

  • Query (Q)

  • Key (K)

  • Value (V)

  • Dot-product attention

  • Scaling by √d (p.51–52)

  • Matrix form: softmax(QKᵀ / √d) V

5. Self-Attention

  • Q, K, V all come from the same sequence (p.55–59).

  • Allows each token to attend to all others in the layer.

6. Transformers

The central innovation (p.60):

All recurrence is removed. Everything uses attention.

7. Multi-Head Attention

  • Multiple attention mechanisms run in parallel (p.61–63).

  • Each head specialises in different linguistic relationships.

8. Positional Encoding

Essential because Transformers have no recurrence:

  • Sinusoidal position encodings (p.73–83).

  • Preserve relative distance and absolute position.

9. Full Transformer Architecture

Pages 84–97 show the classic encoder/decoder diagrams:

  • Encoder stack:

    • Input embeddings + positional encoding

    • Multi-head self-attention

    • Feed-forward network

    • Residual connections + layer norm

  • Decoder stack:

    • Masked self-attention (p.100–101)

    • Encoder–decoder attention

    • Feed-forward

    • Residuals + normalisation

10. Transformers vs RNNs

(p.107–121)

Transformers:

  • Parallelisable

  • Handle long-range dependencies

  • Better performance

  • Require careful training (learning-rate warm-up on p.120)

RNNs:

  • Sequential

  • Slow

  • Struggle with long contexts

⭐ Key Takeaways

1. Attention solves RNN bottlenecks

  • Removes the “single vector” bottleneck.

  • Allows long-range dependencies.

2. Self-attention is the core building block

  • Each word attends to all others in the sequence.

  • Enables parallelisation.

3. Multi-head attention allows multiple linguistic perspectives

  • Heads specialise (syntax, coreference, alignment, etc.).

4. Transformers eliminate recurrence

  • Dramatically faster training.

  • Superior performance across NLP tasks.

5. Positional encodings give Transformers a sense of order

  • Without them, attention is permutation-invariant.

6. Decoder uses masked self-attention

  • Prevents looking at future words during training.

7. Training Transformers is harder

  • Requires warm-up

  • Layer norm stabilisation

  • Deep architectures with many parameters

✅ 1. Lecture 3 — Text Classification

CSC1110 Lecture 3 Text Classifi…

🔍 Deep Summary

This lecture introduces text classification as a supervised learning problem: supplying labelled text examples to train a classifier capable of predicting categories for unseen texts.

The Classification Pipeline

  • Input → Feature Extraction → Classifier → Output
    (implied by examples throughout the lecture)

  • Common tasks: spam detection, sentiment analysis, topic classification.

Bag-of-Words (BoW) Representation

  • Words are treated as unordered features.

  • Leads to high-dimensional, sparse vectors.

  • Cannot capture word order, but simple & strong baseline.

TF-IDF Weighting

  • Elevates important words, penalises common ones.

  • Represented mathematically as:
    tf(t, d) × idf(t)
    (definition shown in slide tables contrasting raw counts vs tf-idf values).

Naive Bayes Classifier

  • Uses Bayes’ rule to compute class probabilities.

  • Assumes conditional independence between features.

  • Works surprisingly well on text due to sparsity and high dimensionality.

Logistic Regression vs Naive Bayes

  • NB is generative (learns p(x, y)).

  • LR is discriminative (learns p(y | x)).

  • LR often outperforms NB when training data is large.

Evaluation Metrics

  • Accuracy unreliable under class imbalance.

  • Emphasis on precision, recall, F1.

⭐ Key Takeaways

  • TF-IDF > BoW for most classification tasks.

  • NB is fast but limited by independence assumption.

  • LR handles correlated features better.

  • Always check class imbalance → prefer F1, not accuracy.

✅ 3. Lecture 2 — N-Gram Language Modelling

CSC1110CSC1122 Lecture 2 N-Gram…

🔍 Deep Summary

A language model (LM) assigns probabilities to sequences of words.

Core Idea

Use n-grams to approximate the probability of sequences using the Markov assumption:

P(wₙ | w₁…wₙ₋₁) ≈ P(wₙ | wₙ₋₁ … wₙ₋₍ₙ₋₁₎)

Issues with N-grams

  • Data sparsity grows exponentially with n.

  • Smoothing is essential.

Smoothing Techniques

  • Laplace smoothing: simple but suboptimal.

  • Good-Turing: redistributes probability mass to unseen events.

  • Kneser-Ney: state-of-the-art for n-gram LMs (best at continuation probabilities).

Evaluation

  • Perplexity (PP):
    PP = 2^H
    Lower is better.

⭐ Key Takeaways

  • N-grams make LM computation tractable.

  • Smoothing is required → prevents zero probabilities.

  • Perplexity compares LMs.

  • N-grams fail at long-range dependencies → neural LMs needed.

  • Chain rule is used to compute gradients.

  • Gradients passed backward from output to hidden to input.

  • Weight update rule:
    w := w – η ∂L/∂w

Problems

  • Overfitting → solved with dropout, regularisation.

Hidden Markov Models (HMMs)

Pages 15–33 explain:

  • Markov assumption

  • Decoding via Viterbi algorithm

Training examples (p.25–26) show estimation via relative frequencies.

Problems with HMMs

(p.34–38)

  • Only uses previous tag + current word.

  • Cannot use rich linguistic features.

Conditional Random Fields (CRFs)

(p.39–44)

  • Discriminative sequence model.

  • Can incorporate:

    • prefixes

    • suffixes

    • word shape

    • context window

  • Decoded with Viterbi but trained via SGD.

⭐ Key Takeaways

  • HMMs = generative, limited context.

  • CRFs = discriminative, flexible features.

  • Viterbi → most likely tag sequence.

  • Real NLP tasks (NER, POS) benefit from CRFs.
